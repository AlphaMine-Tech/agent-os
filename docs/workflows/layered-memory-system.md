# Layered Memory System

## Memory Layers
- live session context = immediate reasoning only
- daily notes (`memory/YYYY-MM-DD.md`) = raw recent residue
- curated long-term memory (`MEMORY.md`) = active durable truth
- decision records (`docs/decisions/*.md`) = rationale and tradeoffs
- project/workflow docs (`docs/projects`, `docs/workflows`) = stable knowledge

## Core Rules
- every durable fact gets one canonical home
- transcripts are fallback, not primary memory
- daily notes are append-only raw context
- MEMORY.md should stay small, current, and trustworthy
- move stable project knowledge into docs
- use decision docs for important choices that may need revisiting

## Retrieval Model
- search structured files first
- read only relevant excerpts
- avoid relying on transcript archaeology when structured docs can answer the question
